档案页定位

首页只给历史样本一眼判断，档案页负责把“后来发生了什么”完整摆出来。

中文投资站里最重要的信任感，往往不是来自一句推荐理由，而是来自你愿不愿意把成功、失效、止盈、止损都摆出来。当前档案页已经具备这样的能力，这版 demo 只是把阅读顺序和信息层级整理得更明确。

信任建立页

不是只看赚了多少，而是看当时为什么推、后来为什么成、为什么失效、为什么止盈止损。

比历史列表更重要的是复盘结构

这页应该先给结果，再给原因，再给版本变化和来源说明。用户不是来浏览一串旧记录，而是想判断这套推荐体系是否值得长期继续看。所以首页里的历史样本预览，只能作为入口，完整逻辑必须回到这里看。

完整样本 3 条 版本对照 来源标识 累计收益 / 超额

首页与档案页分工

首页只保留样本摘要，档案页才展示完整时间线、版本轨迹、来源说明和失效原因。

这页最重要的不是收益

收益只是结果的一部分，更重要的是能不能看到“为什么这么走”和“处理是否到位”。

刻意不做

不把档案页改成空泛的荣誉墙，也不把所有历史版本压回首页。

失效原因

风险边界

高风险样本更适合强调止损执行和仓位纪律，而不是单次结果。

档案价值

这类样本能告诉用户系统如何面对错误判断，比只展示成功案例更重要。

档案卡内部结构建议

按“结果 -> 理由 -> 版本 -> 来源 -> 时间线”排，比把所有字段揉成一张卡更符合中文阅读习惯。

信息层 当前已有能力 建议放置位置
结果层 状态、累计收益、超额 卡头先给判断入口
原因层 reason / explanationSummary / consensus 卡中段重点说明
变化层 versionDiff / versionHistoryItems 单独做版本对照块
来源层 sourceNote / benchmarkSource 紧跟在表现区，解决信任问题

筛选器应该放哪

状态筛选、来源筛选、权限提示更适合放右侧固定区，不要压在时间线主阅读区上方抢主位。

来源说明为什么要长期保留

“已验证 / 预估 / 模拟 / 待说明”这些标签不是装饰，而是复盘可信度的核心部分。

权限与 CTA

游客先看部分历史样本，注册用户继续比较成功与失效样本，会员再解锁完整档案、来源和更多版本轨迹。

这页不能退化成什么

只要退化成下面这些形态，档案页的价值就会被削弱。

不能只剩收益榜

如果只给涨跌排名，用户就看不到理由、版本和处理过程。

不能只保留成功案例

失效和止损样本同样重要，它们决定用户是否信任你的风险控制。

不能把所有深内容塞回首页

首页只负责导流，完整档案链路仍应留在档案页。
