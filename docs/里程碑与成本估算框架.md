# 里程碑与成本估算框架（v0.1）

本文提供阶段拆分与成本估算口径，用于项目排期与预算规划。成本口径为人月（PM/研发/测试/设计/算法/运维）。

**1. 里程碑拆分**

**M1：PC Web + 股票推荐 + 实名 + 站内消息**
- 目标：实现核心推荐能力与实名闭环
- 范围：登录/实名、股票推荐、市场动态、站内消息、Admin 基础

**M2：国内期货策略 + 套利模块**
- 目标：上线期货策略与套利推荐
- 范围：期货策略/套利详情、合约行情、风控提示

**M3：会员体系 + 运营报表**
- 目标：商业化闭环与运营监控
- 范围：会员产品、订单、转化漏斗、推荐效果统计

**2. 人员配置（参考）**
- 产品经理 1
- 后端 2
- 前端 2（PC Web + Admin）
- 算法/策略 1-2
- 测试 1
- 运维/数据工程 1（可兼职）

**3. 工期与人月估算（参考）**

**M1（8-10 周）**
- 产品 1.0 人月
- 前端 2.0 人月
- 后端 2.5 人月
- 算法 1.5 人月
- 测试 1.0 人月
- 运维 0.5 人月
- 合计：8.5 人月

**M2（6-8 周）**
- 产品 0.8 人月
- 前端 1.5 人月
- 后端 2.0 人月
- 算法 1.5 人月
- 测试 0.8 人月
- 运维 0.4 人月
- 合计：7.0 人月

**M3（6-8 周）**
- 产品 0.8 人月
- 前端 1.5 人月
- 后端 2.0 人月
- 算法 0.8 人月
- 测试 0.8 人月
- 运维 0.4 人月
- 合计：6.3 人月

**4. 成本范围估算方法**
- 成本 = 总人月 * 人月单价
- 人月单价根据团队地域与资历调整
- 可将算法与数据成本单独计入（付费数据费用、算力费用）

**5. 风险与缓冲**
- 数据源接入周期与授权不确定
- 期货策略上线需更多合规与风控审查
- 建议预留 15%-20% 缓冲
