# 阶段6：会员转化与内容节奏

最后更新: 2026-03-18
状态: Done

## 阶段目标

建立游客、注册用户、会员的内容分层和固定运营节奏，让推荐网站从“可访问”升级为“可持续增长”。

## 核心结果

- 清晰的游客/注册/会员权益边界
- 首页与详情页 CTA 统一
- 固定内容更新节奏上线
- 会员先开通后 KYC 的激活边界落地
- 后台运营节奏检查台升级为可编辑任务流
- 实验看板可继续按设备、用户阶段、实验组趋势拆解

## 权益分层建议

游客：
- 看今日主推荐
- 看部分资讯与部分历史档案

注册用户：
- 看完整推荐逻辑
- 收藏关注
- 查看更多历史档案

会员用户：
- 看完整推荐池
- 看盘中更新
- 看完整复盘与更深风险说明

## 内容节奏建议

- 08:30 今日重点推荐
- 11:30 午盘变化提醒
- 15:30 收盘复盘
- 周末 下周关注方向

## 主要改动点

- 优化首页与推荐页 CTA 文案
- 调整会员引导与注册引导逻辑
- 结合现有会员状态能力做最小分层

## 设计原则

- 不用粗暴拦截取代价值证明
- 先让用户感知价值，再推动注册与会员转化
- 内容节奏本身就是留存机制

## 验收标准

- 页面上有明确的转化动作
- 权益边界清晰，不自相矛盾
- 内容发布节奏稳定可执行
- 前后端相关流程可跑通

## 风险点

- 如果权益边界过早过硬，会影响拉新
- 如果免费内容过多，又会削弱转化

## 偏差说明

- 本轮先做前台转化表达，不提前做复杂会员拦截或多层付费墙
- 会员分层现阶段统一以后端派生 `activation_state + kyc_status + member_level` 为依据，暂未继续扩展更细会员实验分组
- 真实节奏入口先复用现有页面路由，不提前承诺不存在的“午盘独立发布后台”或“实时推送中心”
- 实验分析本轮继续采用轻量事件表 + 聚合看板，不提前做复杂漏斗系统、队列消费或多维归因
- 最终转化事件本轮先复用真实订单成功回调与后台改单动作，不提前做更复杂的多触点归因模型
- 多跳归因本轮先补首页 / 资讯页 / 历史档案页 / 关注页 / 策略页到会员页的通用链路，不提前扩成任意页面、任意中间跳转都自动串联的复杂 journey 引擎
- 当前实验分组仍以轻量单分组 `default` 记录页面来源，先验证来源价值，再决定是否在这些入口继续做 A/B 分流
- 看板拆解本轮先基于 experiment_events 做轻量聚合，不提前做订单级唯一去重、复杂 cohort 或更重的可视化图表组件

## 遗留问题

- 更复杂的 cohort 分析仍留在后续专题，不并入阶段6验收
- 订单级唯一去重仍留在后续专题，不并入阶段6验收
- 任意页面任意中间跳转的通用 journey 引擎仍留在后续专题，不并入阶段6验收
- 如果后续要做更强后台能力，还可以把检查台升级成真正可提交、可追责的发布任务表
- 当前后台已经补上来源页面 / 日期趋势 / 支付渠道拆解，但还没有继续下钻到“设备 / 用户阶段组合 / 实验组趋势对比”
- 当前 pending -> promotion 只覆盖资讯页与历史档案页，若后续首页、关注页、更多运营入口也要做登录后承接，需要抽成通用 journey 规则
