# 客户端页面原型字段清单（增长模块）

## 1. 浏览历史页 `/history`
- 筛选字段
- `content_type`：`ALL|STOCK|FUTURES|NEWS`
- `date_range`：开始时间、结束时间
- 列表字段
- `id`
- `title`
- `content_type`
- `content_id`
- `source_page`
- `viewed_at`
- 操作字段
- `delete_item`
- `clear_all`

## 2. 充值记录页 `/wallet/recharge-records`
- 筛选字段
- `status`：`PENDING|PAID|FAILED|CANCELLED`
- `date_range`
- 列表字段
- `id`
- `order_no`
- `amount`
- `pay_channel`
- `status`
- `paid_at`
- `remark`
- 操作字段
- `export_recent_3_months`

## 3. 分享推广页 `/invite`
- 顶部卡片字段
- `invite_code`
- `share_url`
- `channel`
- `status`
- `expired_at`
- 统计字段
- `share_click_uv`
- `invite_register_count`
- `invite_verified_count`
- `invite_first_pay_count`
- 邀请记录列表字段
- `invite_record_id`
- `invitee_user_id_masked`
- `status`
- `register_at`
- `first_pay_at`

## 4. 分享奖励页 `/invite/rewards`
- 概览字段
- `reward_balance_cash`
- `reward_balance_coupon`
- `reward_balance_vip_days`
- 奖励明细字段
- `reward_id`
- `reward_type`
- `reward_value`
- `trigger_event`
- `status`
- `issued_at`
- `invitee_user_id_masked`
- 规则说明字段
- `daily_cap`
- `monthly_cap`
- `risk_notice`

## 5. 交互要求
1. 所有列表支持分页与空态。
2. 状态字段统一用枚举映射中文文案。
3. 邀请用户信息默认脱敏展示。
4. 奖励状态为 `FROZEN|REJECTED` 时展示原因文案。

## 6. 埋点建议
- `share_link_copy_click`
- `share_link_open_click`
- `invite_page_view`
- `reward_page_view`
- `history_item_open`
- `recharge_record_export_click`
