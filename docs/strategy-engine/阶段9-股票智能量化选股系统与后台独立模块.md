# 阶段9：双资产智能策略研究平台 V2

最后更新: 2026-03-22  
状态: Done

> 说明：本文件沿用原阶段9文档路径，但当前真相源已经从“股票智能量化选股第一轮”升级为“股票 + 期货双资产研究平台 V2”。路径保留仅为避免引用漂移，不再代表 stock-only 口径。

## 阶段目标

把阶段8完成的真实数据底座继续升级为一个可研究、可运行、可审核、可复盘、可图谱追踪、可兼容旧消费链路的双资产研究平台。

阶段9最终收口目标是：

- Admin 形成两个一级模块：`智能选股`、`智能期货`
- `strategy-engine` 输出股票与期货双资产研究产物
- `strategy-graph` 成为 run 级图快照和子图查询的内部图服务
- Go 后端承接双资产 profile / run / review / publish / evaluation 闭环
- Client 继续使用旧消费契约，并通过共享 helper 统一 explanation、评估和版本差异体验

## 最终架构

### 服务与职责

- Go 后端：多源数据接入、上下文构建、任务调度、持久化、图谱代理、Admin API、旧消费链路兼容
- `strategy-engine`：股票/期货研究链、结构化 evidence、报告、发布 payload、版本与解释增强字段
- `strategy-graph`：图快照写入、快照读取、按实体查询子图；支持 Neo4j 或内存降级
- Admin：双资产研究工作台与审核发布入口
- Client：股票 / 期货 explanation 与 version-history 的统一消费层

### 当前双资产模块口径

- 一级菜单：
  - `智能选股`
  - `智能期货`
- 当前最终页面口径：
  - 总览
  - 运行中心
  - 策略模板
  - 策略配置
  - 规则
  - 因子
  - 图谱
  - 角色
  - 候选与组合
  - 评估复盘
  - 审核发布

### 图谱口径

- 图服务已独立为 `services/strategy-graph`
- Admin 不直接访问图服务，统一走 Go 代理接口
- run 图谱失败时采用降级语义：
  - run 和发布链不丢
  - `graph_write_status` 可标识失败
  - 图谱页可提示无图快照或读取失败

## 本阶段已完成能力

### 1. 股票与期货双资产 run 闭环

- 股票与期货都已具备：
  - profile / version / template
  - run / compare / candidate / portfolio
  - review / approve / reject / publish
  - evaluation / leaderboard
  - published snapshot 固化
- 当前明确补齐的对称性缺口是期货 profile version 专用查询链路，现已与股票一致开放

### 2. 研究链与结构化产物

- 股票主链已支持：
  - `market_regime`
  - universe -> seed_pool -> candidate_pool -> portfolio
  - evidence cards / related entities / memory feedback
  - graph snapshot 写入与 graph summary 回读
- 期货主链已支持：
  - 市场状态判定
  - evidence / related entities / memory feedback
  - graph snapshot 写入与结构化解释
  - evaluation 回填与榜单聚合

### 3. 双资产后台研究平台

- `智能选股` 与 `智能期货` 两个一级菜单已接入导航与权限体系
- 双资产页面都已统一走 Go 接口，不直接解析 Python 原始运行时结构
- 旧 `MarketCenter` 仍保留迁移期入口和跳转能力，不删除旧操作路径

### 4. 客户端统一 explanation helper

- `client/src/lib/strategy-version.js` 已成为股票/期货 explanation 的统一真相源
- 当前统一消费的增强字段包括：
  - `market_regime`
  - `portfolio_role`
  - `graph_summary`
  - `evidence_cards`
  - `related_entities`
  - `evaluation_meta`
  - `version_diff`
- 老 recommendation / strategy 记录缺少新字段时，仍保持 fallback 可读

## 接口与类型口径

阶段9最终对外口径固定为：

- 股票管理路由：`/api/v1/admin/stock-selection/*`
- 期货管理路由：`/api/v1/admin/futures-selection/*`
- 图谱代理路由：`/api/v1/admin/strategy-graph/*`

双资产 run / explanation 当前要求稳定返回的增强字段包括：

- `market_regime`
- `graph_snapshot_id`
- `graph_summary`
- `related_entities`
- `memory_feedback`
- `evaluation`
- `published_portfolio_snapshot` / `published_contract_snapshot`

所有新增字段均走“只增不改”，不删除旧消费字段与旧 fallback。
