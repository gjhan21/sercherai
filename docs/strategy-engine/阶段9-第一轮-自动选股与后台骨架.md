## 第一轮目标

第一轮只做最小可感知闭环，不同时推进全部研究能力。

第一轮目标固定为：
- Python 在 `AUTO` 模式下可不依赖人工 `seed_symbols` 自动完成股票池构建、候选池生成和最终 Top N 选择
- Admin 新增独立一级菜单 `智能选股`
- Admin 至少具备 `总览 / 运行中心 / 策略配置 / 候选与组合` 四个页面骨架
- 配置、运行、候选查看、审核发布形成最小闭环
- 图谱增强和多角色推演先保留接口和轻量接入点，不在第一轮做重平台

## 第一轮不做什么

- 不做全量金融图谱管理台
- 不做完整多世界推演控制台
- 不做复杂历史回测平台
- 不做策略 profile 的大量预置模板
- 不做独立图数据库或重型图谱基础设施

## 第一轮默认收口口径

- 第一轮只上线一个默认 profile：`profile_default_stock_auto`。
- 第一轮只建设 4 个后台页面：`总览 / 运行中心 / 策略配置 / 候选与组合`。
- `selection_mode` 默认固定为 `AUTO`，Admin 发起 run 时必须显式带出该字段。
- `seed_symbols` 不再作为默认输入；只有 `MANUAL / DEBUG` 模式才读取该字段。
- 发布与审核继续复用现有 `strategy-engine` 发布链路，第一轮不新造第二套发布系统。
- 客户端展示契约保持不变；第一轮的发布结果继续回写现有推荐表和发布归档。
- 图谱、事件、多角色在第一轮只做轻量增强和摘要证据，不做独立研究控制台。
- 旧 `MarketCenter` 股票量化入口在迁移期先保留，不因为第一轮上线而强行移除。

## 第一轮任务协议默认口径

当前股票任务 payload 需要从“人工种子模式”升级成“自动 / 人工双模式”。

新增字段固定为：
- `selection_mode`
- `universe_scope`
- `profile_id`
- `debug_seed_symbols`
- `min_listing_days`
- `min_avg_turnover`

字段口径：
- `AUTO`: 默认模式，Python 自己生成股票池和候选种子
- `MANUAL`: 人工指定股票池或种子，用于运营或研究干预
- `DEBUG`: 调试模式，允许只看少量股票并输出更多中间产物

保留旧字段：
- `seed_symbols`
- `excluded_symbols`
- `limit`
- `max_risk_level`
- `min_score`

但从第一轮开始，`seed_symbols` 只作为 `MANUAL / DEBUG` 使用，不再作为默认主链字段。

### 字段兼容规则

- `selection_mode` 缺省值固定为 `AUTO`。
- `profile_id` 对 Admin 发起 run 为必填；对旧每日生成入口可暂时允许为空，但服务端必须自动回填默认 profile。
- `universe_scope` 是新主字段；`market_scope` 继续保留做兼容 fallback。
- `debug_seed_symbols` 仅在 `DEBUG` 模式生效；`MANUAL` 模式继续使用 `seed_symbols`。
- `min_listing_days` 与 `min_avg_turnover` 进入 `universe_config` 的同时，也允许以顶层入参透传，便于调试和历史重跑。

### Python 任务 payload 示例

```json
{
  "trade_date": "2026-03-21",
  "selection_mode": "AUTO",
  "profile_id": "profile_default_stock_auto",
  "universe_scope": "CN_A_ALL",
  "market_scope": "CN_A_ALL",
  "debug_seed_symbols": [],
  "seed_symbols": [],
  "excluded_symbols": [],
  "limit": 5,
  "lookback_days": 120,
  "min_listing_days": 180,
  "min_avg_turnover": 50000000,
  "max_risk_level": "MEDIUM",
  "min_score": 75
}
```

## 第一轮页面状态流

### 总览页

- 页面进入时先请求 `GET /overview`，不额外拼装多个旧接口。
- 点击“立即运行”后直接调用 `POST /runs`，成功后跳到运行中心并高亮新 run。

### 运行中心页

- 页面初始化时先拉 profile 选项，再拉 runs 列表。
- 打开详情抽屉时单独请求 `GET /runs/:run_id`，避免列表接口塞入大体量明细。

### 策略配置页

- 左侧始终展示 profile 列表与默认标识，右侧展示参数分组表单。
- 版本记录 tab 单独懒加载，避免每次打开页面都拉取全部版本快照。

### 候选与组合页

- 页面进入时先取最近一次成功 run，作为默认 run 选择器。
- 候选池和最终组合走两个并行接口，避免一个大接口承载全部详情。
- 单只股票的因子拆解、理由、风险摘要在右侧详情面板内展示，不再额外跳页。
