## 实际完成范围

本阶段实际采用了“轻图谱 + 金融场景 agent panel”的最小实现：
- 股票与期货都支持 `bull / base / bear / shock` 四类场景卡片输出，每个场景都带 thesis、分数调整、动作建议和风险信号
- agent panel 统一聚合趋势、事件、流动性、风险、基差五类角色意见，输出单标的共识动作
- 风险 agent 保留否决权，高风险样本会直接进入 `REJECT`
- 图谱不做持久化，只输出适合现阶段开发节奏的 `graph_summary` 文本摘要
- 现有 job 协议与报告骨架保持不变，阶段4仅以附加字段增强解释层

## 验收结果

- 同一股票或期货合约已可输出多场景解释结果
- 多角色 agent 观点已可收敛为统一 `consensus_action`
- 高风险期货样本可被风险 agent 否决，相关测试已覆盖
- 本地测试通过：`pytest`，6 个用例全部通过

## 偏差说明

- 阶段4只做“金融场景版多代理”，不做大规模 agent 社会模拟
- 图谱本轮只保留聚合摘要，不落库存储，也不引入独立图数据库
- agent 数量控制在五类核心角色，先验证解释层价值，不提前扩展复杂协商流程

## 遗留问题

- 当前场景引擎仍基于规则与内置样本，尚未接入真实行情、事件和产业链关系数据
- agent 投票仍是轻量规则融合，尚未接入可配置权重、回测校准与长期记忆
- 图谱仍是摘要文本，若阶段5之后要做复盘穿透，仍需再评估结构化存储方案
