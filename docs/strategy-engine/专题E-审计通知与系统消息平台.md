# 专题E：审计通知与系统消息平台

最后更新: 2026-03-23  
状态: Planned

## 1. 专题目标

把当前散落在审核、发布、任务中心和 workflow 的消息能力，升级为统一的审计通知与系统消息平台。

## 2. 当前基线

- 当前已有 Review Center、System Jobs、Workflow Messages、User Messages 等多个消息入口
- 发布、回放、作业运行、默认策略落库、数据治理等关键行为尚未统一进入同一事件流
- 部分对象已有本地 snapshot 与审计信息，但语义不统一、入口分散

## 3. 专题完成标准

- 默认策略自动落库、强制发布、驳回、回滚、truth 重建失败、provider 切换、研究运行失败等都进入统一事件流
- 后台支持通知路由、去重、聚合、升级和确认闭环
- 审核中心、任务中心、流程消息开始统一消费 audit event summary
- 支持研究事件、数据事件、发布事件、运营事件四类统一模型

## 4. 非目标

- 第三方 IM 全渠道编排
- 对外 webhook 市场
- 全量 SOC / SIEM 替代系统

## 5. 推荐第一轮范围

1. 统一 audit event schema
2. 审计事件入库与统一写入入口
3. 通知路由与聚合策略
4. Admin 消息中心第一版
5. 现有消息入口回读统一事件流
