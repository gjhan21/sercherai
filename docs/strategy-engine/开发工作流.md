# Strategy Engine 开发工作流

最后更新: 2026-03-22
状态: 生效中

## 目标

确保 `strategy-engine` 后续开发严格遵循 Vibe Coding 节奏，避免边做边散、做完无记录。

## 开发前动作

每次开始开发前，必须完成以下检查：

1. 阅读 `docs/strategy-engine/README.md`
2. 阅读当前阶段文档
3. 确认本次只做当前阶段范围内事项
4. 确认验收标准与回写要求
5. 如有范围变更，先改文档，再写代码

## 开发中动作

- 优先把输入、处理、输出链路打通，不提前扩成复杂研究平台
- 每一轮只做一个最小可感知闭环
- 同时关注三件事：策略价值、系统边界、可验收性
- 若实现与阶段文档不一致，必须先更新文档再继续
- 先保证股票 / 期货输出结果可解释，再考虑复杂代理与长期记忆
- 阶段9进入实现期时，优先让 Python 自主生成股票池与候选池，再扩图谱增强和多角色推演
- Admin 侧如新增智能选股能力，优先收口为独立模块，不再散落到旧页面和旧菜单
- 阶段9如还处于 Planned 状态，必须先同时收口主方案文档和第一轮蓝图，不允许把关键口径留给实现过程中临场决定
- 涉及新旧字段并存时，必须先在文档中写清兼容策略、默认值和回退口径，再开始编码

## 开发后动作

每次完成一个可提交节点后，必须回写：
- 完成了什么
- 哪些地方与计划不一致
- 当前默认约定或还遗留哪些问题
- 下一阶段是否可开始

建议在对应阶段文档中补充以下内容：
- 实际改动文件
- 实际完成范围
- 验收结果
- 风险与后续建议
- 兼容策略与回归命令

## 阶段状态定义

- `Planned`: 已规划，未开始
- `In Progress`: 正在开发
- `Blocked`: 有依赖或风险，暂时阻塞
- `Done`: 功能已完成并完成文档回写
- `Deferred`: 暂缓执行，等待后续阶段

## 范围控制原则

以下事项不应在前几阶段提前扩张：
- 大规模 agent 社会模拟
- 自建复杂图数据库
- 实时高频策略
- 自动实盘交易
- 完整研究平台重构
- 把社交媒体式重型多世界仿真直接照搬进金融主链

优先级更高的是：
- 服务骨架稳定
- 股票与期货 MVP 可跑通
- 输出结构可解释
- 发布适配与复盘闭环清晰
- Python 先真正掌控股票自动选股主链，再扩展研究级图谱和角色系统

## 文档更新模板

可在每个阶段文档末尾保持以下结构：
- 当前状态
- 已完成
- 偏差说明
- 默认约定或遗留问题
- 进入下一阶段条件

## 阶段9历史约束

以下规则用于追溯阶段9第一轮 stock-only 启动时的默认约定；当前阶段9已经收官，如继续迭代请优先以当前 README 与阶段9主文档为准。

- 当开发进入阶段9第一轮时，额外遵守以下规则：
- 先读主方案文档，再读第一轮蓝图，缺一不可
- 第一轮正式开工前，再读一遍启动清单，按任务包和验收闸门推进
- 第一轮不得把 `智能选股` 能力继续散落在 `MarketCenterView`
- 第一轮不得把完整图谱平台、完整回测平台和重型多世界控制台混入同一轮
- 第一轮所有对象、接口、权限都走“只增不改”兼容策略
