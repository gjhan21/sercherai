# 阶段8：真实数据接入与真相源增强

最后更新: 2026-03-22  
状态: Done

## 阶段目标

让 `strategy-engine` 从“依赖内置样本和局部快照可运行”升级为“默认消费 Go 后端统一真相源、具备稳定数据新鲜度摘要、并能在前后台解释链路中复用同一套上下文”。

阶段8的核心不再是继续做页面，而是把研究输入、发布复盘、上下文解释和后台数据状态统一收口到同一套真实数据主链。

## 本阶段完成结果

### 1. 真实上下文主链完成切换

- 股票主链已默认通过 Go 后端上下文 API 构建 `stock-selection` 输入，不再默认依赖 `DEFAULT_MARKET_SEEDS`
- 期货主链已默认通过 Go 后端上下文 API 构建 `futures-strategy` 输入，不再默认依赖 `DEFAULT_FUTURES_SEEDS`
- 股票与期货上下文都保留显式 fallback，但 fallback 必须带 warning，并且优先级严格低于真实真相源

### 2. 统一真相源与数据摘要完成收口

- 股票与期货 report / explanation / Admin run detail 已统一携带 `context_meta`
- 关键稳定摘要统一收口到：
  - `selected_trade_date`
  - `price_source`
  - `warnings`
  - `graph_summary`
  - `graph_snapshot_id`
- Admin 概览与详情不再依赖运行期临时拼装结果，开始稳定消费本地归档与上下文摘要

### 3. 期货真实研究输入完成增强

- 期货上下文已补齐真实行情、成交额、持仓、期限结构、全曲线斜率、跨期价差分位
- 期货仓单/库存已进入标准表与同步链路，支持品种级与结构级摘要
- 仓单结构已覆盖仓库、区域、品牌、产地、等级等解释维度
- 期货 explanation / role review / basis 理由摘要已能直接复用这些真实字段

### 4. Admin 数据源与数据质量闭环已可用

- 市场中心与数据源页已可查看同步结果、质量摘要、真相源重建和问题日志
- 股票 / 期货同步链路可以完成“同步 -> 入标准表 -> 构建上下文 -> 生成任务”的端到端链路
- 作业中心和研究模块能稳定展示真实数据依赖状态，而不再只靠远端运行时临时结果

### 5. 旧消费路径保持兼容

- 旧每日股票生成入口、旧期货生成入口继续保留
- explanation / version-history / publish record / replay 继续兼容旧客户端主消费契约
- 阶段8收官不要求同步改旧客户端接口，只做增强字段追加与真相源稳定化

## 本阶段验收结论

阶段8按以下口径完成验收：

- 股票与期货上下文默认优先真实真相源，受控 fallback 仍带 warning
- Admin 侧可稳定展示数据新鲜度、来源摘要与任务依赖提示
- 股票与期货都已跑通“同步数据 -> 构建上下文 -> 生成作业 -> 发布/入库 -> explanation 可读”的闭环
- 文档状态、接口、页面消费与当前实现一致

## 明确不并入阶段8验收的后续专题

以下内容不再继续并入阶段8，而是保留为后续专题：

- 更大的多源市场数据治理与供应商抽象平台
- 更细的股票事件图谱、资讯语义与主题/行业关系建模
- 更重的期货研究因子、交割地事件、品牌/等级深层画像
- 专门的默认策略自动落库审计通知或系统消息平台
- 更重的库存、仓单、跨品种 spread 与多市场联动研究平台

## 默认约定

- `strategy-engine` 继续只消费内部统一真相源，不直接绑定外部供应商
- 旧入口与旧消费契约继续保留，直到新模块稳定后再讨论收缩
- 阶段8后续如再增强真实数据能力，统一按专题文档推进，不再回退到阶段8状态
