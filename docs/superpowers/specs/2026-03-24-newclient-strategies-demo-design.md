# New Client Strategies Demo Design

## 背景

本页是 `newclient` 全站 demo 中第一批页面之一，用于建立首页之后的第二个核心锚点。

首页已经确认“推荐主位 + 焦点研报双主线”的风格，`strategies` 页必须继续承接这个方向，把首页的“今天先看什么”进一步展开成“为什么推荐、怎么执行、边界在哪里”。

## 页面主任务

`strategies` 页的任务不是列资讯，也不是做后台筛选台，而是要回答以下问题：

- 当前推荐是什么
- 为什么成立
- 怎么执行
- 风险边界在哪里
- 股票、期货、市场事件之间如何分层阅读

## 设计边界

本页只能使用当前真实能力可承接的内容：

- 股票推荐
- 股票 insight / performance / version history 的表达性承接
- 期货策略与 guidance 承接
- 市场事件承接
- 与关注页、资讯页、档案页的跳转关系

本页不允许新增以下能力：

- 大盘实时指数条
- 热门题材榜
- 社区讨论
- 直播、短视频
- 假 K 线或假实时分时图
- 与当前 `client` 不匹配的新后端模型

## PC 页面结构

PC 端定位为“中文金融站 / 推荐工作台”。

固定结构如下：

1. 顶部导航与页面定位
2. 主推荐 Hero
3. 推荐理由与执行边界
4. 股票 / 期货 / 市场事件分层区
5. 右侧阅读顺序与页面关系说明
6. 底部站点信息区

### 说明

- 股票主推荐始终是第一主位
- 期货和市场事件是承接能力，不抢股票主位
- 右侧固定区负责告诉用户“应该怎么读、下一步去哪”

## H5 页面结构

H5 端定位为“国内金融 App / 观点流 + 详情流”。

固定结构如下：

1. 顶部信息头
2. 栏目 / 过滤切换
3. 推荐主卡
4. 推荐理由卡
5. 风险边界卡
6. 观点流列表
7. 期货 / 市场事件承接卡
8. 底部 Tab

### 说明

- H5 不照搬 PC 工作台，而是压成单列阅读链
- 仍然要保持“先看结论，再看理由，再继续刷观点”的节奏
- 期货和事件卡必须明显属于策略页，而不是像资讯列表

## 首页如何导流过来

本页的首页入口固定来自以下位置：

- 首页主推荐 CTA
- 首页“查看完整观点”
- 首页期货机会轻承接卡
- 首页中与推荐主线相关的二级入口

因此本页首屏必须延续首页的语言，不能像换了一个产品。

## 与其他页面的关系

本页与以下页面关系最紧密：

- `home`：负责导入“今天先看什么”
- `watchlist`：承接“加入关注后怎么跟踪”
- `news`：承接“与推荐有关的研报和资讯深读”
- `archive`：承接“历史样本和处理结果”

页面中必须保留去这几页的自然入口说明。

## 不能做成什么样

本页不允许做成以下样子：

- 只有股票列表，没有策略解释
- 后台筛选台风格
- 被资讯流稀释的伪推荐页
- 期货与事件完全抢掉主推荐主位
- 看完本页后不知道下一步去哪里

## 视觉原则

- 与首页保持同一套蓝金金融系
- PC 保持研究终端感，信息密度适中但不粗糙
- H5 保持国内金融 App 感，单列节奏明确
- 不做欧美数据大屏风
- 不做紫色 AI SaaS 风
