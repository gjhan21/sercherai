# New Client Archive Demo Design

## 背景

`archive` 页承接首页的历史样本预览和策略页的复盘入口，是“过去推荐了什么、结果如何、为什么处理”的完整复盘台。

本页必须同时展示成功与失败样本，强调历史理由、涨跌结果、止盈止损原因和版本变化，而不是只展示收益结果。

## 页面主任务

`archive` 页要回答：

- 过去推荐过哪些样本
- 这些样本后来涨跌如何
- 当时的推荐理由与后续处理原因是什么
- 版本变化和结论变化如何影响复盘

## 设计边界

本页只使用当前真实能力：

- 历史推荐列表
- 状态筛选
- 来源标签
- 历史表现与复盘
- 版本差异
- 版本轨迹

本页不能新增：

- 纯收益排行榜
- 只展示成功案例
- 模拟回测图或虚构交易收益曲线

## PC 页面结构

PC 端定位为“历史推荐复盘台”。

固定结构：

1. 档案定位区
2. 筛选与概览
3. 历史推荐时间线
4. 样本复盘卡
5. 版本差异与变化说明
6. 右侧复盘方法与阅读顺序

### 说明

- 首屏先说明这页看什么、怎么读
- 样本复盘必须同时包含正向和反向案例
- 版本差异是这页的核心，不是附属说明

## H5 页面结构

H5 端定位为“单列复盘页”。

固定结构：

1. 页面头与筛选条
2. 焦点历史样本卡
3. 历史样本流
4. 单条样本详情
5. 历史理由、涨跌结果、后期处理原因、版本变化

### 说明

- H5 不做成绩单式排行页
- 单条详情必须讲清“理由 -> 结果 -> 后续处理”
- 作为二级页面，不放主底部 Tab

## 页面关系

- `home`：历史样本预览导流
- `strategies`：从“今天为什么看”回到“过去怎么做”
- `watchlist`：从持续跟踪回到历史版本对照
- `news`：看与历史样本对应的资讯背景

## 不能做成什么样

- 只有涨跌结果，没有理由和处理说明
- 成功样本陈列页
- 只有时间线，没有复盘结论
- 做成涨跌榜或回测战报

## 视觉原则

- 蓝金金融系下强调“档案 / 复盘”气质
- PC 像研究终端中的复盘工作台
- H5 像国内金融 App 的单列档案阅读页
- 正文信息更重，装饰更轻
