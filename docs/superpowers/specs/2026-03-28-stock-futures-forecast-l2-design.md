# 股票与期货预测增强 L2 设计规范

## Summary

`L2` 的目标是在 `L1` 的研究编排和记忆回灌基础上，增加“轻量金融图谱 + 多情景走势推演”能力，让系统不只会解释当前推荐，还能更可信地推演未来路径。

本阶段仍不进入重型深度模拟，重点是把“后面怎么走”做成结构化、可验证、可复盘的情景系统。

## Stage Goal

`L2` 重点解决三个问题：

1. 当前推荐为什么成立之外，未来最可能怎么演化。
2. 未来路径不是一句“看涨 / 看跌”，而是多情景结构。
3. 推演结果要和现有推荐解释、版本历史、复盘体系兼容。

## Core Deliverables

### 1. 轻量金融知识图快照

本阶段不构建通用大图谱，只建立服务于推荐推演的轻量金融关系图。

股票域最小关系集合建议覆盖：

- 公司
- 行业
- 主题
- 上下游链条
- 事件
- 政策
- 资金线索

期货域最小关系集合建议覆盖：

- 合约
- 品种
- 主连 / 主力 / 近月关系
- 产业链
- 库存 / 供需
- 基差 / 价差
- 宏观 / 政策事件

### 2. 多情景推演引擎

本阶段正式引入三层情景：

- 牛情景
- 基准情景
- 熊情景

每个情景最少要包含：

- thesis
- trigger
- confirmation signal
- invalidation signal
- expected window
- action suggestion

### 3. 多角色意见与共识 / veto

本阶段不做真人设模拟，但可以做金融角色函数化意见系统。

股票建议角色：

- 量价角色
- 资金角色
- 行业 / 主题角色
- 风险控制角色
- 事件 / 新闻角色

期货建议角色：

- 方向角色
- 供需角色
- 基差 / 结构角色
- 宏观角色
- 风控角色

每个角色给出：

- stance
- confidence
- summary
- veto flag

最终形成：

- consensus action
- veto reason
- scenario confidence spread

### 4. 市场环境感知模板

本阶段将推演模板与市场环境绑定。

至少区分：

- 风险偏好强
- 风险偏好弱
- 震荡轮动
- 事件驱动

在不同市场环境下，相同标的和相同逻辑的推演路径可以不同。

## Implementation Changes

### 1. 构建轻量关系快照层

数据来源优先复用现有：

- related entities
- related events
- theme / sector tags
- supply chain notes
- market events
- news signals

目标不是构建大而全图谱，而是形成“推演够用”的关系快照。

### 2. 正式引入 scenario package

在 explanation 中新增更稳定的 scenario 包结构。

要求：

- 股票和期货共用骨架
- 每个资产类型有自己的模板
- 结果可进入 insight 和 version history

### 3. 引入角色函数意见层

角色意见来自当前上下文、L1 证据分层和关系快照，而不是随机生成。

重点要求：

- 每个角色的输入来源可解释
- 角色间允许分歧
- veto 机制能真正影响最终结论

### 4. 推演结果写入复盘链

`L2` 新增的情景推演不能只出现在当前详情页，还要能进入后续回看。

至少需要支持：

- 历史版本查看当时的情景推演
- 复盘时比较实际结果和当时情景
- 标记哪类情景长期偏乐观或偏保守

## Public APIs / Interfaces / Types

### 允许的非破坏性扩展

- explanation 中新增更完整的 `scenarios`
- 新增 `scenario_meta`
- 扩展 `agent_opinions`
- 扩展 `version-history` 返回项

### 保持不变

- 现有接口路径
- 现有鉴权和访问链路
- 现有推荐 / 策略发布主流程

## Dependencies

`L2` 依赖 `L1` 先完成以下基础：

- 子问题研究编排
- 失效理由分层
- 记忆回灌
- 轻量置信度校准

没有这些基础，`L2` 的情景推演会变成“更好看的文本”，而不是更可靠的系统能力。

## Test Plan

### 结构验证

- 股票和期货都能输出完整三情景
- 每个情景都有 trigger / invalidation / action
- 角色意见能形成共识或 veto

### 质量验证

- 不同市场环境下模板输出有差异
- 相关关系变化会影响情景结构
- 历史版本可以回看旧情景

### 回归验证

- 现有 explanation 旧字段不回退
- 现有前台即使未读取新字段也不报错

## Out Of Scope

- 不新增独立 analysis 页作为本阶段前提
- 不做外部深度模拟引擎接入
- 不做大规模长期记忆图数据库
- 不做社交舆情世界推演
- 不做全市场全量标的异步重模拟

## Exit Criteria

满足以下条件可认为 `L2` 设计达标并可进入后续实现：

1. 股票和期货都有明确的三情景推演模型
2. 情景具备触发、确认、失效和动作建议
3. 多角色意见与 veto 机制有清晰边界
4. 结果可进入现有 insight 与历史复盘链
