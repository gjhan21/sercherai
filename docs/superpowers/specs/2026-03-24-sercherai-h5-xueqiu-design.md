# SercherAI H5 App 化重构设计

**日期：** 2026-03-24
**范围：** `/Users/gjhan21/cursor/sercherai/client/src/apps/h5`

## 目标

将现有 `/m/*` 页面从“桌面网页缩窄版”重构为更接近中国金融内容 App 的 H5：以内容流、阅读节奏、底部 Tab 和固定 CTA 为核心，不改 backend API，不改 shared auth/session/http 协议。

## 产品方向

- 参考对象：雪球的信息流结构与内容优先节奏
- 产品气质：更专业、更克制，偏投顾/券商内容端，不做社区讨论感
- 首页形态：头条式“今日核心观点”，不做重 Hero 大卡，也不做宫格入口
- 导航结构：`首页 / 资讯 / 策略 / 会员 / 我的`
- 内容关系：`资讯` 为主阅读阵地，`策略` 是精选观点流，`会员` 偏收银台，`我的` 偏账户中心

## 结构原则

### 1. H5 壳层
- 顶部不再使用桌面化品牌栏；改为更轻的移动内容头部
- 底部 Tab 固定且更像 App：图标、标签、活跃态明显，热区更大
- 页面容器不再依赖大块卡片堆叠，而是更像内容流、列表和吸顶区块

### 2. 首页 `/m/home`
- 首屏只做“今日核心观点 + 一句话摘要 + 相关标的/风险标签”
- 紧接相关资讯和延展内容，不做多个并列功能块
- 操作只保留少量高优先级按钮：看策略、看资讯、会员升级/登录

### 3. 资讯页 `/m/news`
- 顶部采用栏目切换 + 搜索，下面直接进入文章流
- 列表像内容 App，不像网页式卡片展示
- 详情页保持正文优先，底部固定操作区承接会员与相关策略

### 4. 策略页 `/m/strategies`
- 改为“精选观点流”，像雪球精选判断，不像策略工作台
- 列表项强调标题式结论、风险、更新时间和一句理由
- 详情按单列阅读：核心结论、理由、风险、事件、下一步动作

### 5. 会员页 `/m/membership`
- 改成移动收银台：套餐、推荐方案、购买动作优先
- 权益说明压缩为短条目，不再占首屏
- 支付 / 实名 / 激活收拢为一条转化链路

### 6. 我的页 `/m/profile`
- 改为账户中心：账户头部、会员状态、消息、订单、实名、邀请等入口
- 历史记录和次级明细下沉，不堆满首屏

## 视觉语言

- 白底 + 灰阶 + 深蓝为主，金色仅做会员与重要状态点缀
- 强调标题层级、信息密度、留白、分组和列表节奏
- 减少大面积渐变玻璃卡；重点放在阅读感、秩序和单手操作
- 支持 `prefers-reduced-motion`

## 技术边界

- 保持现有 H5 router、shared auth/session/http、backend API 不变
- 重构重点在：H5 layout、view 层、专属组件样式和数据映射方式
- PC 入口不受影响，`/m/auth` 继续保持独立全屏认证页
