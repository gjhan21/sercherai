# New Client Auth Demo Design

## 背景

`auth` 页是全站权限链的统一身份入口。它不做营销，不做第三方登录广场，而是强调“快速登录 / 注册 + 邀请码可见 + 登录后回原阅读位置”。

## 页面主任务

`auth` 页要回答：

- 我是从哪里被带到登录页的
- 现在应该登录还是注册
- 邀请码和来源回跳是否仍然保留
- 登录后会回到哪里

## 设计边界

本页只使用当前真实能力：

- 登录
- 注册
- 邀请码绑定
- `redirect` 回跳
- `guestOnly` 入口控制

本页不能新增：

- 第三方登录
- 社交账号绑定页
- 长篇营销介绍

## PC 页面结构

PC 端定位为“高效率身份入口”。

固定结构：

1. 顶部来源说明
2. 登录 / 注册双栏主区
3. 当前支持能力说明
4. 登录后回跳说明

### 说明

- 登录和注册必须并列出现
- 邀请码和 redirect 始终可见
- 页面重心是完成进入，不是做品牌营销

## H5 页面结构

H5 端定位为“移动身份入口”。

固定结构：

1. 顶部返回与品牌区
2. 登录 / 注册单列主区
3. 邀请码状态区
4. 来源回跳说明
5. 当前支持能力说明

### 说明

- H5 采用上下堆叠，不做复杂切屏
- 来源和邀请码都要在首屏附近
- 作为身份页，不放主底部 Tab

## 页面关系

- `news`：正文和附件权限锁触发登录
- `membership`：下单或激活前要求登录
- `profile`：未登录用户先进入身份页
- `home`：邀请码入口默认回首页

## 不能做成什么样

- 社交登录广场
- 强营销注册页
- 登录后丢失来源回跳
- 把邀请码藏到二级说明

## 视觉原则

- 保持蓝金金融系，但更强调安全、清晰与高效率
- PC 双栏明确，H5 单列紧凑
- 表单表达专业克制，不做花哨互动
