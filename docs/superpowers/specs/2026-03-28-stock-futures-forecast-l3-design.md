# 股票与期货预测增强 L3 设计规范

## Summary

`L3` 的目标是面向少量高价值标的，引入“深度模拟 + 长期学习”能力，形成真正差异化的预测增强层。

这不是全量替代现有推荐系统，也不是让所有标的都进入重型推演，而是构建一个高成本、低频次、强审计、强复盘的深推演能力层。

## Stage Goal

`L3` 主要解决两个问题：

1. 对关键标的进行比 `L2` 更深的未来演化推演。
2. 把深度推演结果沉淀为长期记忆和研究资产，而不是一次性文本。

## Core Deliverables

### 1. 选择性深度模拟框架

`L3` 只面向少量高价值对象触发，例如：

- 高权重股票推荐
- 高风险高收益期货策略
- 用户主动发起的高价值单标的分析
- 管理员指定研究任务

触发原则：

- 不对全市场全量跑
- 必须有明确优先级和成本控制

### 2. 金融专用角色库

如果引入更深的模拟或多角色互动，不应沿用通用社交人格，而应采用金融角色函数化设计。

股票可考虑的角色：

- 产业链研究员
- 资金经理
- 事件驱动研究员
- 风险官
- 宏观观察者

期货可考虑的角色：

- 供需分析师
- 套保视角
- 投机资金视角
- 宏观与政策角色
- 风险官

### 3. 异步推演任务与报告资产

`L3` 建议通过异步任务执行，输出：

- 推演任务状态
- 推演报告
- 研究日志
- 中间证据
- 最终观点

这样后续才能：

- 审计
- 复盘
- 对比不同版本
- 判断哪些推演方式更有效

### 4. 长期记忆回灌

`L3` 要把深度推演中的结论、失败点、冲突点和后验结果回写成长期记忆。

长期记忆至少要支持：

- 标的级
- 模板级
- 市场环境级
- 角色级

### 5. 预测质量校准体系

`L3` 不只是多做推演，还要评估推演质量。

建议建立的指标包括：

- 情景命中率
- 触发条件命中率
- 失效条件提前识别率
- 过度乐观 / 过度保守偏差
- 不同角色在不同环境下的有效性

## Implementation Changes

### 1. 引入异步深推演编排器

该编排器负责：

- 选择是否进入深推演
- 组织输入材料
- 调度外部或内部深推演逻辑
- 生成可追踪产物

### 2. 研究日志与工具调用审计

`L3` 的核心价值之一是“可解释的深推演”，因此不能只保存最终结论。

需要保留：

- 输入上下文摘要
- 中间研究步骤
- 工具调用记录
- 角色意见分歧
- 最终汇总理由

### 3. 深度结果与主链融合

即使有深度推演结果，也不应替换主推荐链，而应作为增强层。

建议融合方式：

- 主推荐继续由现有推荐体系产出
- 深度推演给出增强解释、风险补充和未来路径
- 当前台没有深推演结果时，可优雅降级到 `L2`

## Public APIs / Interfaces / Types

本阶段可能新增异步任务与报告相关接口，但前提是：

- 不破坏现有推荐 / insight / version-history 契约
- 深推演能力作为增强能力独立存在
- 允许前台按权限读取深推演结果

## Dependencies

`L3` 依赖：

- `L1` 的研究编排与记忆回灌
- `L2` 的情景推演与角色函数骨架

缺少前两阶段，`L3` 只会成为高成本演示系统，而不是稳定的预测增强层。

## Test Plan

### 任务链验证

- 深推演任务可以创建、执行、完成、失败回收
- 可生成最终报告与中间日志
- 无结果时可回退到 `L2`

### 质量验证

- 高价值标的的深推演结果可审计
- 推演后的记忆可以进入长期学习层
- 后验结果能够反哺角色和模板校准

### 成本控制验证

- 全量市场不会误触发深推演
- 高成本能力具备明确准入规则

## Out Of Scope

- 不做全量标的实时深推演
- 不做无限制开放的通用聊天预测
- 不做和现有推荐链完全平行的第二套生产系统

## Exit Criteria

满足以下条件可认为 `L3` 设计成熟：

1. 深推演只对高价值对象触发
2. 结果具备日志、证据和可审计性
3. 推演输出能回灌长期记忆
4. 质量指标可衡量，不是只看生成文本好不好看
