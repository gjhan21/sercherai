# New Client News Demo Design

## 背景

`news` 页是首页焦点研报和策略深读的承接页，它的任务不是把所有内容摊平，而是把“导读 -> 正文 -> 附件 -> 权限链”整理成顺畅的阅读路径。

本页必须强化研报的独立存在感，同时保留当前真实的分类、搜索、正文、附件和权限承接逻辑。

## 页面主任务

`news` 页要回答：

- 今天最值得读的焦点研报是什么
- 当前栏目下有哪些文章值得继续读
- 正文、附件和权限状态如何承接
- 游客、登录、VIP 分别能读到哪里

## 设计边界

本页只使用当前真实能力：

- 分类切换
- 资讯域内关键词搜索
- 文章列表
- 正文详情
- 附件列表与下载承接
- 游客 / 登录 / VIP 权限链

本页不能新增：

- 全站搜索
- 内容瀑布流推荐
- 社区讨论、评论、弹幕
- 假专题站或资讯热榜

## PC 页面结构

PC 端定位为“导读进入深读的正文页”。

固定结构：

1. 资讯页定位与今日焦点研报
2. 分类切换区
3. 主列表区
4. 正文详情区
5. 附件与权限承接区
6. 右侧栏目、权限、阅读链说明

### 说明

- 研报、新闻、期刊必须视觉分层
- 正文详情必须明显大于资讯列表的权重
- 右侧区域负责解释栏目意义和权限链，而不是堆内容

## H5 页面结构

H5 端定位为“移动阅读链入口”。

固定结构：

1. 顶部标题、搜索、分类切换
2. 焦点研报卡
3. 主文章卡
4. 内容流列表
5. 正文详情模式
6. 附件与权限 CTA
7. 底部 Tab

### 说明

- 搜索仍然只作用于资讯域
- 内容流继续保留国内内容 App 的下滑阅读节奏
- 正文、附件、权限承接必须清楚，不做纯摘要卡

## 页面关系

- `home`：承接焦点研报和资讯导流
- `strategies`：承接与推荐相关的深读解释
- `archive`：承接历史样本与复盘背景
- `membership`：承接正文全文和附件权限升级
- `auth`：承接登录后继续原阅读位置

## 不能做成什么样

- 只剩文章列表、没有正文区
- 把研报、新闻、期刊做成一个语气
- 做成内容瀑布流或热门榜单
- 丢失游客 / 登录 / VIP 的阅读链

## 视觉原则

- 蓝金金融系中突出“研报重点感”
- PC 像中文金融站的深读页，不像资讯门户
- H5 像国内金融内容客户端，不像 H5 宣传页
- 权限提示要专业克制，不做营销大 banner
