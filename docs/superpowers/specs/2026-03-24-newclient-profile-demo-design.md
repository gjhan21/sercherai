# New Client Profile Demo Design

## 背景

`profile` 页在当前 `client` 中本质上是账户管理台，不是轻量资料编辑页。它需要让用户先看到今天要处理什么，再继续查看会员、实名、消息、邀请和订阅等账户服务。

## 页面主任务

`profile` 页要回答：

- 当前账户身份和会员状态是什么
- 今天需要先处理哪些待办
- 订单、消息、浏览、订阅、邀请等信息去哪里确认
- 下一步最常用的账户动作是什么

## 设计边界

本页只使用当前真实能力：

- 用户资料
- 会员状态 / quota
- 订单
- 浏览历史
- 站内消息
- 订阅
- 分享链接与邀请记录
- 实名提交

本页不能新增：

- 社交主页
- 单纯头像昵称编辑页
- 后台查询台式的强工具语气

## PC 页面结构

PC 端定位为“账户管理台”。

固定结构：

1. 账户身份概览
2. 今日行动板
3. 待办中心
4. 查询中心
5. 快捷入口
6. 右侧账户摘要与待处理事项

### 说明

- 先看今天要做什么，再看模块查询
- 查询中心仍可保留多模块切换，但文案语气改为“确认 / 管理”
- 右侧必须持续显示会员、实名、未读消息状态

## H5 页面结构

H5 端定位为“我的 / 账户中心”。

固定结构：

1. 账户主卡
2. 今日待办
3. 常用入口宫格
4. 账户服务卡流
5. 消息 / 邀请 / 实名区块
6. 底部 Tab

### 说明

- 首屏必须看到账户状态和关键待办
- 卡片流顺序按“最常处理的动作”组织
- 底部 Tab 使用：首页 / 策略 / 关注 / 资讯 / 我的

## 页面关系

- `membership`：查看升级、订单和激活后的结果
- `auth`：未登录用户先完成身份进入
- `news`：回到未完成的阅读链
- `watchlist` / `strategies`：回到主要内容使用链

## 不能做成什么样

- 只有资料编辑表单
- 没有待办和状态的空壳“我的”页
- 像后台运营查询台
- 会员状态、实名状态和消息提醒被藏起来

## 视觉原则

- 与会员页保持同一套账户体系观感
- PC 更像管理台，H5 更像金融 App 的“我的”
- 用蓝金主体系区分重要动作，用语义色区分待办状态
