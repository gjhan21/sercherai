# New Client Watchlist Demo Design

## 背景

`watchlist` 是首页与策略页之后的第三个阅读锚点，用来承接“决定继续跟踪后，用户下一步怎么回访”。

这页必须严格建立在当前 `client` 已有的本地关注与变化跟踪能力上，不得被误画成云端自选股、社交组合或实时行情中心。

## 页面主任务

`watchlist` 页必须回答四个问题：

- 我重点关注的对象最近发生了什么变化
- 哪些变化值得优先回访
- 这些变化分别属于状态、资讯、风险边界还是结论变化
- 下一步应该去策略、资讯还是档案继续读

## 设计边界

本页只使用当前真实能力：

- 本地关注列表
- 最新状态变化
- 资讯变化
- 风险边界变化
- 结论变化
- 版本 / 时间线对照
- 快速移除

本页不能新增：

- 云端同步自选股
- 社区共享组合
- 实时盘口或行情大屏
- 排行榜、热榜、聊天室

## PC 页面结构

PC 端定位为“变化工作台”。

固定结构：

1. 页面定位与高优先回访样本
2. 关注主列表
3. 变化拆层说明
4. 右侧回访节奏、能力边界、页面关系
5. 底部站点信息区

### 说明

- 首屏先看“今天先回访谁”，不是先看长表格
- 主列表里必须同时体现最新结论与变化摘要
- 右侧区域负责告诉用户如何回访，以及为什么这页不同于行情页

## H5 页面结构

H5 端定位为“单列回访页”。

固定结构：

1. 页面头与最近更新时间
2. 今日优先回访卡
3. 变化速览卡组
4. 关注列表流
5. 去策略 / 资讯 / 档案的下一步入口
6. 底部 Tab

### 说明

- H5 不做行情小组件堆叠
- 每条关注卡仍然先给结论，再给变化原因
- 底部 Tab 使用：首页 / 策略 / 关注 / 资讯 / 我的

## 页面关系

- `home`：告诉用户今天先看什么
- `strategies`：解释为什么推荐、怎么执行
- `watchlist`：告诉用户加入关注后怎么持续跟踪
- `news`：承接“变化背后的研报和资讯”
- `archive`：承接“历史样本怎么复盘”

## 不能做成什么样

- 只有名称和涨跌的轻量列表
- 没有变化摘要的假关注页
- 像自选行情 App 的盯盘页
- 像社区组合页或多人共享池

## 视觉原则

- 延续首页与策略页的蓝金金融系
- PC 保持“工作台”感，但信息秩序清晰
- H5 保持国内金融 App 的跟踪阅读节奏
- 正文区不写设计解释，边界说明只放在 demo 入口页
